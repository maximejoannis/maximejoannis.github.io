Contexte et stratégie d’investigation

Une démarche QA structurée pour comprendre la technologie réelle de rendu, identifier les limites des outils desktop classiques et choisir une approche industrialisable.

Contexte

Application desktop moderne basée sur Electron

AMN AUDITOR combine une fenêtre Windows native avec un moteur Chromium embarqué. L’interface métier est rendue sous forme HTML/CSS/JavaScript, et non comme une interface Windows classique.

ElectronChromiumReactMaterial UIWindows Desktop
Objectif QA

Évaluer la faisabilité

Identifier une stratégie robuste permettant de couvrir les parcours utilisateur principaux sans accès direct au code source.

Méthode

Stratégie d’investigation en 5 étapes

  1. Lancer l’application desktop.
  2. Identifier la technologie réelle de rendu.
  3. Tester l’accès UIA / WinAppDriver.
  4. Extraire le DOM Chromium.
  5. Définir la solution industrialisable.

Scénario automatisé cible

Ouvrir un projet existant

Précondition : AMN AUDITOR est installé localement et WinAppDriver ou DevTools est disponible selon l’approche choisie.

Analyse QA et recommandation finale

L’automatisation doit viser le DOM applicatif plutôt que l’arbre UIA Windows, car les composants métier sont encapsulés dans Chromium.

Recommandation

Architecture hybride DevTools + Python/Pytest

La recommandation est d’industrialiser les tests avec une architecture hybride : lancement contrôlé de l’application, connexion DevTools sur le port 9222, pilotage DOM par JavaScript, assertions via Pytest et génération de rapports d’exécution.
