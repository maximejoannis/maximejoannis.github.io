Sur mon dépôt GitHub portfolio QA, j’ai conçu un pipeline CI/CD déclenché à chaque push ou pull request pour exécuter automatiquement une suite Robot Framework, générer les rapports et archiver les preuves.

Objectif & Périmètre

  • Déclencher la CI à chaque push/PR
  • Exécuter 20 tests Robot Framework
  • Générer automatiquement les rapports
  • Archiver artefacts + preuves (captures/logs)
  • Rendre les runs auditables et comparables
« Une CI utile, c’est une CI reproductible, traçable et exploitable. »

Implémentation (pipeline + workflow + qualité)

Réalisation, enchaînement d’étapes, et pratiques pour une CI fiable.

Workflow GitHub Actions

  • Création d’un workflow YAML déclenché sur push et pull request.
  • Organisation des jobs/steps pour une lecture “pro” (setup → run → reports → artefacts).

Setup automatique de l’environnement

  • Installation Python + dépendances Robot Framework.
  • Préparation des prérequis d’exécution UI (driver/grid selon le run).

Exécution de la suite (20 tests)

  • Lancement Robot Framework en pipeline avec statut CI (success/fail).
  • Logs exploitables pour diagnostic en cas d’échec.

Reporting & Artefacts

  • Génération automatique : output.xml, log.html, report.html.
  • Upload des artefacts (rapports + preuves : captures/logs).

Workflow d’exécution (lecture rapide)

  1. Push/PR → déclenchement GitHub Actions
  2. Setup Python + dépendances
  3. Démarrage/connexion au runtime UI (grid si activé)
  4. Exécution Robot Framework (20 tests)
  5. Génération rapports + collecte preuves
  6. Upload artefacts → consultation dans l’onglet Actions

Bonnes pratiques CI (qualité)

  • Environnement standardisé (reproductibilité)
  • Traçabilité : rapports + preuves attachées au run
  • Feedback rapide sur PR (régression visible)
  • Exécution indépendante du poste local
  • Diagnostic facilité via logs/artefacts
